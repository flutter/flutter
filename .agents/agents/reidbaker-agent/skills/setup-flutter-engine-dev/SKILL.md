# Skill: Provision Flutter Engine & Framework Development Environment

## Context
This skill provisions a fully compliant, trunk-based Flutter development environment from scratch. It utilizes direct Git checkouts (bypassing tarballs) and bootstraps the Chromium engine C++ toolchain (`gclient`, `ninja`, `gn`).

## Prerequisites
- A Linux host environment.
- Git and Curl installed.

---

## Execution Steps

### 1. Define the Workspace
Choose a stable directory to house the toolchains. Avoid temporary agent scratch spaces.
```bash
export WORKSPACE="$HOME/Documents/flutter_dev"
mkdir -p "$WORKSPACE"
cd "$WORKSPACE"
```

### 2. Install Host Toolchain (Linux)
The Flutter desktop framework requires system-level compilation headers.
```bash
sudo apt-get update
sudo apt-get install -y clang cmake libgtk-3-dev ninja-build pkg-config
```

### 3. Provision Chromium `depot_tools`
The engine build relies on Chromium's custom build tooling.
```bash
cd "$WORKSPACE"
git clone https://chromium.googlesource.com/chromium/tools/depot_tools.git
export PATH="$WORKSPACE/depot_tools:$PATH"
```

### 4. Clone the Unified Flutter Repository
The engine code is now unified within the main framework repository.
```bash
cd "$WORKSPACE"
git clone https://github.com/flutter/flutter.git
```

### 5. Fetch Engine Toolchains (`gclient`)
Configure and fetch the massive dependency graph (Dart SDK, Skia, Vulkan, etc.).
*CRITICAL: You must append `--no-history`. Fetching the full Git history for massive dependencies like `icu` will routinely cause network timeouts and deadlock the setup.*
```bash
cd "$WORKSPACE/flutter"
cp engine/scripts/standard.gclient .gclient
gclient sync --no-history
```

### 6. Headless Android SDK Bootstrap
Bypass Android Studio entirely. Install the headless CLI and force the strict SDK versions demanded by the `master` channel.
```bash
export ANDROID_HOME="$HOME/Android/Sdk"
mkdir -p "$ANDROID_HOME/cmdline-tools"
cd "$ANDROID_HOME/cmdline-tools"

wget -qO cmdline-tools.zip "https://dl.google.com/android/repository/commandlinetools-linux-14742923_latest.zip"
unzip -q cmdline-tools.zip

# Enforce the strict 'latest' directory structure required by modern sdkmanager
rm -rf latest
mv cmdline-tools latest
rm cmdline-tools.zip

export PATH="$PATH:$ANDROID_HOME/cmdline-tools/latest/bin"

# Accept all proprietary licenses automatically
yes | sdkmanager --licenses

# Install platform-tools and the specific SDK versions required by Flutter master
sdkmanager "platform-tools" "platforms;android-36" "build-tools;28.0.3"

# Bind Flutter to the SDK
export PATH="$WORKSPACE/flutter/bin:$PATH"
flutter config --android-sdk "$ANDROID_HOME"
yes | flutter doctor --android-licenses
```

### 7. Final Diagnostics
Run `flutter doctor` to trigger the final framework artifact downloads and validate the environment.
```bash
flutter doctor
```

## Permanent Environment Configuration (`~/.zshrc` / `~/.bashrc`)

Append the following to the shell configuration file to permanently persist the environment.

```bash
# Chromium depot_tools (Must be prepended)
export PATH="$HOME/Documents/flutter_dev/depot_tools:$PATH"

# Flutter Framework CLI
export PATH="$HOME/Documents/flutter_dev/flutter/bin:$PATH"

# Flutter Engine Tool ('et')
export PATH="$HOME/Documents/flutter_dev/flutter/engine/src/flutter/bin:$PATH"

# Android SDK
export ANDROID_HOME="$HOME/Android/Sdk"
export PATH="$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools"

# ----------------------------------------------------
# Crucial Performance Optimizations for Massive Repos
# ----------------------------------------------------
# 1. Turn on caching to prevent severe lag in autocomplete
zstyle ':completion:*' use-cache on
zstyle ':completion:*' cache-path ~/.zsh/cache

# 2. Ignore Python cache artifacts generated by gclient/gn
zstyle ':completion:*' ignored-patterns '*?.pyc' '__pycache__'

# 3. Disable automated interactive prompts (Oh-My-Zsh) that deadlock background tasks
zstyle ':omz:update' mode disabled
DISABLE_AUTO_UPDATE="true"
```
